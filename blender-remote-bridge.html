Blender Remote Bridge 部署文档

在 AWS 上使用 Claude Code 远程控制公司内网 Mac 上的 Blender 3.6，用于插件开发和调试。

一、系统架构

Mac 无法接受入站连接（公司防火墙），但可以发出 HTTP 请求。采用拉模式：Mac 主动轮询 AWS 获取任务。

AWS                                      Mac 公司内网

┌───────────┐  POST   ┌────────────┐         ┌──────────────┐      ┌──────────┐
│  CLI Tool │───────▶│ Relay Server│◀──poll──│ Bridge Agent │─────▶│ Blender │
│ (cli.py)  │◀───────│  (FastAPI)  │────────▶│ (bridge.py)  │◀─────│  Addon   │
└───────────┘  GET    └────────────┘         └──────────────┘      └──────────┘
                         :8080                                    localhost:9876

数据流

Claude 调用 CLI → CLI 发任务到 Relay → Mac Bridge 轮询拿到任务 → 调 Blender Addon 执行 → 结果原路返回

延迟预估

操作预估往返说明
截图1-3 秒含 1s 平均轮询延迟 + 截图 + 传输
脚本执行1-2 秒简单脚本；复杂操作视具体执行时间
日志获取1-2 秒直接返回内存缓冲区

三、组件说明

AWS Relay Server — 中继服务器

FastAPI 应用，负责中转任务和结果。内存存储，任务 5 分钟过期自动清理。

方法端点用途
POST/tasks创建任务（CLI 调用）
GET/tasks/pending获取待执行任务（Bridge 轮询）
POST/tasks/{id}/result提交执行结果（Bridge 调用）
GET/tasks/{id}/result?wait=N长轮询等待结果（CLI 调用）
POST/files/{name}上传文件
GET/files/{name}下载文件
GET/health健康检查（无需认证）

Blender Addon — 插件

关键设计：bpy 操作必须在主线程执行。通过 queue.Queue + bpy.app.timers 桥接后台 HTTP 线程和主线程。

模块功能
executor.pyHTTP 线程调用 execute_on_main_thread(code)，放入队列并阻塞等待；定时器回调（100ms）在主线程取出执行
server.pyhttp.server.HTTPServer 后台线程，绑定 127.0.0.1:9876
capture.py截图优先用 macOS screencapture，备选 bpy.ops.screen.screenshot（兼容 3.6 和 4.x）；日志通过替换 stdout/stderr 写入环形缓冲区

Mac Bridge Agent — 轮询代理

每 1 秒轮询 Relay 获取任务，根据类型调用 Blender Addon 对应端点，结果 POST 回 Relay。连接异常自动重试，不会崩溃。

AWS CLI Tool — 命令行

Claude 在 AWS 上使用的命令行工具：

python cli.py screenshot                              # 截图 Blender 窗口
python cli.py exec "bpy.data.objects.keys()"           # 执行 Python 代码
python cli.py exec my_script.py                        # 执行脚本文件
python cli.py logs                                     # 查看控制台日志
python cli.py push plugin.py --dest ~/addons/         # 推送文件到 Mac

四、部署步骤

1

AWS — 启动 Relay Server

# 安装依赖
pip install fastapi uvicorn pydantic

# 启动服务（替换密钥）
cd /opt/workspace/bl/relay
BLENDER_RELAY_API_KEY=your-secret-key \
  uvicorn server:app --host 0.0.0.0 --port 8080
2

Mac — 安装 Blender 插件

# 复制插件到 Blender 3.6 插件目录
cp -r addon/blender_remote \
  ~/Library/Application\ Support/Blender/3.6/scripts/addons/

# 然后在 Blender 中：
# Edit → Preferences → Add-ons → 搜索 "Remote Control" → 勾选启用
启用后 Blender 控制台会输出 [Remote Control] Addon registered - server on localhost:9876
3

Mac — 启动 Bridge Agent

# 安装依赖
pip install requests

# 启动轮询代理（替换 AWS IP 和密钥）
cd bridge
RELAY_URL=http://你的AWS公网IP:8080 \
BLENDER_RELAY_API_KEY=your-secret-key \
  python bridge.py
4

AWS — 使用 CLI 控制 Blender

cd /opt/workspace/bl/cli

# 截图
python cli.py screenshot

# 执行代码
python cli.py exec "print(bpy.data.objects.keys())"

# 查看日志
python cli.py logs

# 推送插件文件到 Mac
python cli.py push my_plugin.py --dest ~/blender_plugins/my_plugin.py

五、快速上手（复制粘贴即可）

以下是从零开始的完整操作命令，按顺序执行。两边的 BLENDER_RELAY_API_KEY 保持一致，你的AWS公网IP 替换为实际 IP。

▶ AWS 端操作

1

创建 conda 环境并安装依赖（首次）

# 创建专用 conda 环境
conda create -n blender-relay python=3.12 -y

# 激活环境
conda activate blender-relay

# 安装依赖
pip install fastapi uvicorn pydantic requests mcp
2

启动 Relay Server

# 激活环境（如果还没激活）
conda activate blender-relay

# 进入项目目录
cd /opt/workspace/bl/relay

# 启动 Relay Server（后台运行）
BLENDER_RELAY_API_KEY=mysecretkey \
  uvicorn server:app --host 0.0.0.0 --port 8080 &

# 确认服务正常
curl http://localhost:8080/health
# 应返回 {"status":"ok","tasks":0,"files":0}

▶ Mac 端操作

3

克隆代码

git clone git@github.com:killinux/bl.git
cd bl
4

安装 Blender 插件

# 复制插件到 Blender 3.6 插件目录
cp -r addon/blender_remote \
  ~/Library/Application\ Support/Blender/3.6/scripts/addons/

# 然后打开 Blender：
# Edit → Preferences → Add-ons → 搜索 "Remote Control" → 勾选启用
# 控制台应输出: [Remote Control] Addon registered - server on localhost:9876
5

启动 Bridge Agent

# 安装依赖
pip install requests

# 启动 Bridge（替换 AWS IP 和密钥）
RELAY_URL=http://你的AWS公网IP:8080 \
BLENDER_RELAY_API_KEY=mysecretkey \
  python bridge/bridge.py

# 看到以下输出表示连接成功：
# Bridge starting - relay=http://xxx:8080 blender=http://127.0.0.1:9876
# Poll interval: 1.0s

▶ 回到 AWS 端使用

6

用 CLI 控制 Blender

cd /opt/workspace/bl/cli

# 截取 Blender 窗口截图
python cli.py screenshot

# 执行 Python 代码
python cli.py exec "print(bpy.data.objects.keys())"

# 查看 Blender 控制台日志
python cli.py logs

# 推送文件到 Mac
python cli.py push my_plugin.py --dest ~/blender_plugins/my_plugin.py

# 执行一个 .py 脚本文件
python cli.py exec my_script.py
启动顺序很重要：AWS Relay → Blender 启用插件 → Mac Bridge → AWS CLI。另外 AWS 安全组需要开放 8080 端口。

六、MCP Server 集成（推荐）

将 Bridge 封装为 MCP Server 后，Claude Code 可以直接用自然语言控制 Blender，无需手动跑 CLI 命令。

配置方法

1

安装 MCP SDK（已在 conda 环境中安装可跳过）

conda activate blender-relay
pip install mcp requests
2

配置 Claude Code

编辑项目目录下的 .claude/mcp.json：

{
  "mcpServers": {
    "blender-remote": {
      "command": "/opt/miniconda3/envs/blender-relay/bin/python",
      "args": ["/opt/workspace/bl/mcp/blender_mcp_server.py"],
      "env": {
        "RELAY_URL": "http://localhost:8080",
        "BLENDER_RELAY_API_KEY": "your-secret-key"
      }
    }
  }
}
// 注意：command 必须用 conda 环境的 python 绝对路径
// 因为 MCP Server 是由 Claude Code 启动的子进程，不会继承 conda activate
3

重启 Claude Code

重启后 Claude 自动加载 MCP 工具，直接用自然语言即可：

# 不再需要手动跑 CLI，直接对 Claude 说：
"截个图看看 Blender 里什么样"
"列出场景里所有物体"
"在原点创建一个红色立方体"
"把 Cube 的材质改成金属质感"
"帮我查一下 Blender 控制台有没有报错"

MCP 工具列表

工具名功能对应 CLI
blender_screenshot截取 Blender 视口截图cli.py screenshot
blender_exec执行任意 Python/bpy 代码cli.py exec
blender_logs获取控制台日志cli.py logs
blender_push推送文件到 Maccli.py push
blender_scene_info获取场景所有对象信息（名称、类型、位置、材质）—
blender_object_info获取单个对象详情（顶点数、包围盒、修改器等）—
MCP vs CLI：MCP 是推荐方式，Claude 自动选择合适的工具调用。CLI 作为备选，也可以在 MCP 不可用时手动使用。两者底层走同一套 Relay 通道。

八、安全说明

项目措施
认证所有端点（除 /health）通过 X-API-Key Header 认证，密钥通过环境变量 BLENDER_RELAY_API_KEY 设置
Blender AddonHTTP 服务仅绑定 127.0.0.1，不暴露到网络
传输加密建议 Relay 前加 nginx 反代配 HTTPS，或 uvicorn 直接配 SSL 证书
代码执行/exec 端点可执行任意 Python，仅用于开发环境，API Key 防止未授权访问

九、故障排除

问题排查方式
Bridge 连不上 Relay检查 AWS 安全组是否开放 8080 端口；检查 RELAY_URL 是否正确
Bridge 报 Blender addon not reachable确认 Blender 已启动且插件已启用；检查 localhost:9876 是否可达
截图超时macOS 首次使用 screencapture 可能需要授予屏幕录制权限：系统设置 → 隐私与安全性 → 屏幕录制
脚本执行无响应脚本可能阻塞了 Blender 主线程，默认 30 秒超时；检查 Blender 控制台输出
CLI 一直等待确认 Bridge 正在运行；检查 Relay /health 端点查看任务数
API Key 错误确认 AWS 和 Mac 两端环境变量 BLENDER_RELAY_API_KEY 一致

十、环境变量汇总

变量名用于默认值说明
BLENDER_RELAY_API_KEY全部dev-key-change-me共享 API 密钥
RELAY_URLBridge / CLIhttp://localhost:8080Relay 服务器地址
BLENDER_ADDON_URLBridgehttp://127.0.0.1:9876Blender Addon 地址
