Blender Remote Bridge 部署文档

在 AWS 上使用 Claude Code 远程控制公司内网 Mac 上的 Blender，基于 blender-mcp 插件。

一、系统架构

Mac 无法接受入站连接（公司防火墙），但可以发出 HTTP 请求。采用拉模式：Mac 主动轮询 AWS 获取任务。

AWS                                         Mac 公司内网

┌───────────┐  POST   ┌────────────┐            ┌──────────────┐      ┌──────────────┐
│  Claude   │───────>│ Relay Server│<───poll────│ Bridge Agent │─────>│ Blender      │
│  Code/CLI │<───────│  (FastAPI)  │───────────>│ (bridge.py)  │<─────│ blender-mcp  │
└───────────┘  GET    └────────────┘            └──────────────┘      │ addon        │
                         :8080                   TCP Socket :9876      └──────────────┘
Claude 调用 CLI/MCP -> Relay Server -> Mac Bridge 轮询拿到任务 -> TCP Socket 发给 blender-mcp -> 结果原路返回

二、项目文件结构

bl/
├── relay/ ── AWS 端，FastAPI 中继服务器
│   ├── server.py ── 服务主程序
│   ├── models.py ── 数据模型
│   └── requirements.txt
├── bridge/ ── Mac 端，轮询代理（TCP Socket 连接 blender-mcp）
│   ├── bridge.py ── 主程序
│   ├── config.py ── 环境变量配置
│   └── requirements.txt
├── cli/ ── AWS 端，Claude 用的 CLI
│   ├── cli.py ── 命令行工具
│   └── requirements.txt
├── mcp/ ── AWS 端，MCP Server（推荐）
│   ├── blender_mcp_server.py ── MCP 工具封装
│   └── requirements.txt
├── aws_server.sh ── AWS 端 Relay 启动/管理脚本
├── mac.sh ── Mac 端 Bridge 启动脚本
└── log/ ── 运行日志（不提交 git）

三、首次搭建

AWS 端

1

创建 conda 环境并安装依赖

conda create -n blender-relay python=3.12 -y
conda activate blender-relay
pip install fastapi uvicorn pydantic requests mcp --root-user-action=ignore
2

启动 Relay Server

conda activate blender-relay
cd /opt/workspace/bl

# 用脚本启动（推荐，自动后台运行 + 写日志）
sh aws_server.sh start

# 或者手动启动
cd relay
BLENDER_RELAY_API_KEY=mysecretkey \
  uvicorn server:app --host 0.0.0.0 --port 8080 &
curl http://localhost:8080/health
3

AWS 安全组开放 8080 端口

AWS 控制台 -> EC2 -> 安全组 -> 入站规则 -> 添加 TCP 8080

Mac 端

4

克隆代码

git clone git@github.com:killinux/bl.git
cd bl
5

安装 blender-mcp 插件

  1. 前往 github.com/ahujasid/blender-mcp 下载 addon.py
  2. 打开 Blender -> Edit -> Preferences -> Add-ons
  3. 点击 Install，选择下载的 addon.py 并启用
  4. 在 3D 视图中按 N 打开侧栏，找到 BlenderMCP 面板
  5. 点击 Start MCP Server，默认监听 localhost:9876
6

安装依赖并启动 Bridge

pip install requests

RELAY_URL=http://你的AWS公网IP:8080 \
BLENDER_RELAY_API_KEY=mysecretkey \
  python bridge/bridge.py

# 看到以下输出表示连接成功：
# Bridge starting - relay=http://xxx:8080 blender=127.0.0.1:9876

四、每天开工（3 步）

1

AWS — 启动 Relay

conda activate blender-relay
cd /opt/workspace/bl
sh aws_server.sh start
2

Mac — 打开 Blender，启动插件

打开 Blender -> N 面板 -> BlenderMCP -> 点 Start MCP Server

3

Mac — 启动 Bridge

cd ~/bl
RELAY_URL=http://你的AWS公网IP:8080 BLENDER_RELAY_API_KEY=mysecretkey python bridge/bridge.py

# 或者直接用脚本：
sh mac.sh

然后在 AWS 上 cd /opt/workspace/bl && claude 启动 Claude Code，直接用自然语言控制 Blender。

五、MCP Server 集成（推荐）

将 Bridge 封装为 MCP Server 后，Claude Code 可以直接用自然语言控制 Blender，无需手动跑 CLI。

配置方法

项目已包含 .claude/mcp.json，指向 conda 环境的 python：

{
  "mcpServers": {
    "blender-remote": {
      "command": "/opt/miniconda3/envs/blender-relay/bin/python",
      "args": ["/opt/workspace/bl/mcp/blender_mcp_server.py"],
      "env": {
        "RELAY_URL": "http://localhost:8080",
        "BLENDER_RELAY_API_KEY": "your-secret-key"
      }
    }
  }
}
// command 必须用 conda 环境的 python 绝对路径
// MCP Server 是 Claude Code 启动的子进程，不会继承 conda activate

重启 Claude Code 后自动加载，直接用自然语言即可。

六、CLI 手动模式（备选）

MCP 不可用时，也可以直接在终端手动使用：

cd /opt/workspace/bl/cli

# 截图
BLENDER_RELAY_API_KEY=mysecretkey python cli.py screenshot

# 执行代码
BLENDER_RELAY_API_KEY=mysecretkey python cli.py exec "print(bpy.data.objects.keys())"

# 执行 .py 脚本文件
BLENDER_RELAY_API_KEY=mysecretkey python cli.py exec my_script.py

# 查看日志
BLENDER_RELAY_API_KEY=mysecretkey python cli.py logs

# 推送文件到 Mac
BLENDER_RELAY_API_KEY=mysecretkey python cli.py push my_plugin.py --dest ~/blender_plugins/my_plugin.py

八、Relay Server API

方法端点用途
POST/tasks创建任务
GET/tasks/pendingBridge 轮询获取待执行任务
POST/tasks/{id}/resultBridge 提交执行结果
GET/tasks/{id}/result?wait=NCLI 长轮询等待结果
POST/files/{name}上传文件
GET/files/{name}下载文件
GET/health健康检查（无需认证）

所有端点（除 /health）通过 X-API-Key Header 认证。内存存储，任务 5 分钟过期自动清理。

九、启动脚本说明

aws_server.sh（AWS 端）

sh aws_server.sh start      # 启动 Relay（后台运行，日志写入 log/relay.log）
sh aws_server.sh stop       # 停止 Relay
sh aws_server.sh restart    # 重启 Relay
sh aws_server.sh status     # 查看运行状态
sh aws_server.sh log        # 查看最近 50 行日志
sh aws_server.sh log 200    # 查看最近 200 行日志

日志文件：/opt/workspace/bl/log/relay.log
PID 文件：/opt/workspace/bl/log/relay.pid

自定义 API Key：默认使用 mysecretkey，可通过环境变量覆盖：
BLENDER_RELAY_API_KEY=你的密钥 sh aws_server.sh start

mac.sh（Mac 端）

# 直接运行，启动 Bridge 轮询代理
cd ~/bl
sh mac.sh

如需修改 AWS IP 或 API Key，编辑 mac.sh 文件。

十、故障排除

问题排查方式
CLI 报 401 UnauthorizedCLI 和 Relay 的 BLENDER_RELAY_API_KEY 不一致
Bridge 连不上 Relay检查 AWS 安全组是否开放 8080 端口；Mac 上 curl http://AWS_IP:8080/health 测试
Bridge 报 Blender addon not reachable确认 Blender 已启动且 blender-mcp 插件已点击 Start MCP Server
任务一直 204 No ContentBridge 没有在运行，或 Bridge 连不上 Relay/Blender
截图超时macOS 首次需要授予屏幕录制权限：系统设置 -> 隐私与安全性 -> 屏幕录制
脚本执行无响应脚本可能阻塞了 Blender 主线程，默认 30 秒超时

十一、环境变量汇总

变量名用于默认值说明
BLENDER_RELAY_API_KEY全部组件dev-key-change-me共享 API 密钥，两端必须一致
RELAY_URLBridge / CLI / MCPhttp://localhost:8080Relay 服务器地址

BLENDER_PORTBridge9876blender-mcp 插件端口
